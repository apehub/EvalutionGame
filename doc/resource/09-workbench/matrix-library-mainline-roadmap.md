# 全库主线路线图

本表用于回答一个问题：如果按“面 -> 线 -> 点”的顺序推进资料库，当前最适合先走哪些主线。

## 使用原则

- 先用这张表确定优先补哪个大面。
- 再进入该面的专属时间页、支系页、事件页和代表卡去拆线。
- 只有当一条线已经能讲清主干逻辑时，才继续往下补点。

## 工作台补强建议

- 若某个面准备进入“补线”阶段，应优先为它补一张 `mainline-reading-order` 或 `topic-entrypoints` 类矩阵。
- 若某个面暂时只有正文，没有入口矩阵，优先先补工具页，再大规模扩正文。
- 若某个面跨越多个目录，先回 `../00-index/surface-map.md` 检查该面的条目是否已能串成一个主题整体。
