# 面级支撑缺口对照表

本表用于回答：当前资料库在“面”这一层面上，缺的究竟是正文，还是导航、来源入口、工作台工具。

## 使用原则

- 若一个主题面缺的是“支撑层”，不要急着继续堆正文。
- 若一个主题面已有正文但难查、难扩、难维护，优先回到本表判断缺的是哪一层支撑。
- 先补入口，再补正文，通常比反过来更稳。

## 当前最缺的支撑动作

### 1. 给偏薄面补“主线阅读顺序”

- 当前只有前寒武纪有较成熟的 `mainline-reading-order`。
- 下一批最值得补的是植物主线、恐龙 / 鸟类主线、大灭绝专题线。

### 2. 给偏薄面补“主题入口反查”

- 当前只有前寒武纪有较明显的 `topic-entrypoints` / `question-phrases` 结构。
- 若后续要真正做到“先看面再看线”，这类入口矩阵需要向更多面复制。

### 3. 给来源层补“按面检索”的跳转提示

- 当前来源层更像通用来源池，而不是“某个大面该先查什么”。
- 后续可考虑按植物、恐龙、无脊椎、人类演化等主题面补来源入口导航。

### 4. 给 registry 补“面级观察视角”

- 当前 `06-registry/` 更偏条目与关系记录。
- 后续如需进一步结构化，可增加按主题面观察全库的 registry 视图或索引映射。
