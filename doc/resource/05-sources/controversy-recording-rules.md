# 争议记录规范

本规范用于统一各条目末尾“争议与说明”部分的写法，避免把不同层级的问题混写在一起。

## 一、什么时候必须写争议

以下情况至少写 1 条争议说明：

- 时间边界在不同来源中存在口径差异
- 系统位置或分类归属存在持续讨论
- 某个机制解释存在多个主流模型
- 著名通俗说法与学界谨慎表述差异较大
- 当前条目使用了综合性总览名称，而非严格自然类群

## 二、推荐结构

每条争议建议尽量包含以下要素：

- 争议点：到底哪一部分存在不同口径
- 主流写法：正文采用哪种表述
- 其他解释：还有哪些常见但未定论的说法
- 证据状态：当前属于 `A` / `B` / `C` 级中的哪一层
- 写作提醒：哪些说法不能写成确定事实

## 三、常见争议类型

### 1. 时间争议

- 例如最早记录时间、某事件起止边界、首次出现年代差异。

### 2. 分类争议

- 例如某化石到底更接近哪个支系、是否属于冠群或干群。

### 3. 机制争议

- 例如寒武纪大爆发、大灭绝成因、生命起源模型。

### 4. 命名与叙事争议

- 例如“鱼类”“无脊椎动物”这类方便组织资料但非严格单系的名称。
- 例如某些通俗叙事把过渡型写成“半成品祖先”的问题。

## 四、写法原则

- 正文优先写主流、较稳妥版本。
- 争议区用于列出差异，不用于堆砌所有少数观点。
- 若证据明显不足，直接写“尚无定论”，不要强行给结论。
- 若争议来自通俗误解而非学界分歧，也应明确标注。

## 五、示例句式

- “关于本条目的最早年代，不同来源存在口径差异；正文采用较广泛接受的区间，较早但争议更大的记录放在本节说明。”
- “本条目使用的是资料整理层面的总览名称，并非严格单系群，阅读时需注意其组织性而非严格分类学含义。”
- “关于该事件成因，目前主流看法倾向多因素叠加，尚无单一决定性解释。”
