# 条目内证据分层模板

本模板用于把“基础事实”“主流解释”“争议点”“待核验项”分开写，避免正文把不同确定性内容混在一起。

## 一、推荐写法

### 基础事实

- 内容类型：时间边界、地理分布、系统位置、首次出现、代表材料
- 优先来源：`A` 级
- 推荐句式：
  - “按 ICS 现行图表，……”
  - “根据 PBDB / Open Tree of Life，……”
  - “教材与数据库对该基础信息口径较一致，……”

### 主流解释

- 内容类型：演化意义、生态位、主要功能解释、主流成因模型
- 优先来源：`A` + `B` 级组合
- 推荐句式：
  - “主流写法通常认为……”
  - “教材、综述和博物馆专题页普遍采用……”

### 争议点

- 内容类型：机制分歧、分类位置不稳、时间上限争议、通俗误读
- 优先来源：`B` + `C` 级组合
- 推荐句式：
  - “关于这一点，现有资料主要有两类解释……”
  - “正文采用较稳妥版本，另一种常见写法见‘争议与说明’。”

### 待核验项

- 内容类型：尚未做数据库交叉验证、尚未补专著、材料信息待核
- 推荐句式：
  - “待补数据库核对”
  - “待补馆藏或原始研究交叉验证”

## 二、适用建议

### 时间主轴页

- 先写 `A` 级年代框架，再写 `B` 级生命史综述，最后才写 `C` 级机制争议。

### 类群页

- 先写系统位置和时间，再写代表衍征和主流生态解释，最后再写分类争议。

### 事件页

- 先分出“事件发生”与“事件为什么发生”。
- 前者可更确定，后者往往需要争议分层。

### 物种卡

- 先写材料、地层和解剖基础事实。
- 再写演化意义。
- 最后写“是不是直接祖先”“行为如何推断”这类争议点。
