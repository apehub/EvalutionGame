# 来源总览

本目录收纳整套资料库的资料入口层。

资料库 2.0 起，本目录逐步收束为“来源入口层”。与来源本体不同的登记、工作台和结构化索引，会逐步迁往：

- `doc/resource/06-registry/`
- `doc/resource/09-workbench/`

## 文件列表

- `web-authoritative.md`：权威网络来源
- `books-core.md`：核心书目与作者体系
- `citation-rules.md`：引用与证据规范
- `controversy-recording-rules.md`：争议与说明部分的统一写法
- `source-registry.md`：高频来源登记册与使用边界（过渡入口，2.0 正式版见 `../06-registry/source-registry.md`）
- `evidence-layer-template.md`：条目内证据分层写法模板
- `../09-workbench/`：工作台工具层（checklist、matrix、问句入口、主题入口等）

## 使用顺序

1. 先用 `web-authoritative.md` 找到条目的主要在线入口。
2. 再用 `books-core.md` 补足学科框架与专题深挖书。
3. 最后按 `citation-rules.md` 给条目安排来源与证据等级。
4. 若条目涉及模型分歧、分类变化或通俗误解，再按 `controversy-recording-rules.md` 统一写“争议与说明”。
5. 若一个来源会反复使用，先登记到 `source-registry.md`；若需要强化条目内部写法，则按 `evidence-layer-template.md` 分层整理。
6. 若要做写作辅助、核查或专题工作台检索，转到 `../09-workbench/README.md`。
7. 若当前要先判断“该从哪个大面开始补资料”，先看 `../00-index/surface-map.md` 与 `../00-index/surface-backlog.md`，再回到本目录找对应来源入口。

## 2.0 迁移提示

- 若要登记高频来源、统一来源 ID 或管理使用边界，优先查看 `doc/resource/06-registry/source-registry.md`。
- checklist、matrix、主题入口和问句入口已迁至 `doc/resource/09-workbench/`。
- 若要先从“面 -> 线 -> 点”推进资料库，优先把 `doc/resource/00-index/surface-map.md`、`doc/resource/00-index/surface-backlog.md` 和 `doc/resource/09-workbench/matrix-library-mainline-roadmap.md` 配合使用。
