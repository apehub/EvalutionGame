# 命名规则

## 目录命名

- 顶层目录使用两位数字前缀，保证长期扩写时顺序稳定。
- 目录名使用英文小写加连字符，例如 `01-timeline`、`04-species-cards`。
- 需要继续细分的目录也使用统一风格，例如 `precambrian`、`vertebrates`、`mass-extinctions`。

## 文件命名

- 综述页使用英文主题名，例如 `origin-of-life.md`、`cambrian.md`。
- 物种卡片优先使用拉丁学名转写，例如 `tiktaalik-roseae.md`、`archaeopteryx-lithographica.md`。
- 类群总览使用现代系统发育或学界常见英文名，例如 `vertebrates.md`、`angiosperms.md`。
- 同一文件只对应一个核心主题，不要把多个时代或多个类群塞进同一页。

## 标题命名

- 一级标题使用中文主名，必要时在正文首段给出英文名、拉丁学名或常见别名。
- 地质年代条目标题建议统一为“中文名（英文名）”。
- 类群与物种条目第一段必须给出系统位置和时间范围，方便快速判断。

## 时间书写

- 默认使用 “Ma” 表示百万年前，使用 “Ga” 表示十亿年前。
- 同一文件内优先统一成阿拉伯数字加单位，例如 `541 Ma`、`2.4 Ga`。
- 时间边界以 ICS 版本为准，如有不同口径，须在“争议与说明”中标出。

## 中文与英文

- 正文以中文为主。
- 首次出现的重要术语、机构名、数据库名、地层名、拉丁学名，应同时给出英文或学名。
- 后续可直接使用中文简称，但不要造成歧义。
